01 Node.js 확인 / 설치 대기

왜? Claude Code는 Node.js 위에서 동작합니다. 모든 설치의 출발점.

1️⃣ 먼저 확인해보세요

터미널 열고 (Win: PowerShell / Mac: Terminal):

node --version
✅ v18.x.x 이상 숫자가 나오면 → 이미 설치됨, 다음 단계로
❌ command not found 같은 에러 → 아래 설치 진행

2️⃣ 설치 (없을 때만)

https://nodejs.org 접속 → LTS(초록 버튼) 다운로드 → 실행 → 모두 Next/Install

⚠️ 설치 후 터미널을 완전히 닫고 새로 열어야 반영됩니다. 안 그러면 여전히 "없음" 뜸.

3️⃣ 다시 확인

node --version → v22.15.0 같은 숫자가 나오면 성공 ✅

02 Git 확인 / 설치 대기

왜? 스킬 다운로드, Dooray MCP 설치, GitHub 사용에 필요해요.

git --version
✅ git version 2.x.x 나오면 OK
❌ 없으면 OS별 설치:
🍎 Mac
git --version # "설치하시겠습니까?" 팝업 → 설치 클릭
🪟 Windows

git-scm.com 접속 → Download for Windows → 다운로드 → 모두 Next → 설치 완료

03 Python 확인 / 설치 대기

왜? Dooray MCP 설치 마법사가 Python을 씁니다. 데이터 분석·대시보드(Streamlit)도 Python 기반.

python3 --version
✅ Python 3.11 이상이면 OK
❌ 없으면 → python.org에서 3.13 LTS 다운로드 → 설치 (Win: "Add to PATH" 체크 필수)

04 Claude Code 설치 대기

이번 강의의 핵심 도구. 한 번 설치하면 다음부턴 claude 한 글자로 시작.

방법 A · npm (Node.js 깔려있을 때)

npm install -g @anthropic-ai/claude-code

1~2분 걸려요. 빨간 글씨(warning) 나와도 무시. 마지막에 added N packages 보이면 성공.

방법 B · 네이티브 설치 (npm 막혔을 때 우회)

🪟 Windows (PowerShell)
irm https://claude.ai/install.ps1 | iex
🍎 Mac / Linux
curl -fsSL https://claude.ai/install.sh | bash

설치 확인

claude --version
✅ 버전 번호 나오면 완료. 막히면 → 왼쪽 🆘 버튼!

💡 권한 팝업이 나오면 Yes 또는 1번 선택. 처음엔 매번 묻는 게 정상.

07 Dooray MCP 설치 과제 제출에 필요 대기

한국어로 "내 두레이 태스크 보여줘"가 동작하게 만들어요. 과제 제출도 이걸로.

1️⃣ API 토큰 발급

  1. nhnent.dooray.com 접속
  2. 우상단 프로필 → 설정 → API 토큰 → 토큰 생성
  3. 토큰 복사 (형식: 조직코드:토큰문자열)
🔒 비밀번호와 동일. 메신저·GitHub에 절대 공유 X

2️⃣ 설치 마법사 실행

git clone https://github.com/ariri48/dooray-mcp.git cd dooray-mcp python setup.py

→ 토큰 입력 → 의존성 설치 → 연결 검증 → MCP 등록까지 자동

3️⃣ 확인

Claude Code 새로 시작 후:

"내 두레이 태스크 보여줘"
✅ 업무 목록 나오면 완료

08 실습 2-A · 엑셀 데이터 분석 대기

사용 파일: 📊 매출_데이터_2026년4월.xlsx (클릭하면 다운로드)

프롬프트

"매출_데이터_2026년4월.xlsx 파일을 읽어서: 1. 전체 매출 합계 2. 팀별 매출 순위 3. 전월 대비 증감이 큰 항목 TOP 3 결과는 '4월_매출분석.md'로 저장해줘"
✅ 바탕화면에 4월_매출분석.md 파일 생성되면 성공

09 실습 2-B · 보고서 자동 생성 대기

2-A의 결과 데이터로 주간보고 작성

"이 데이터로 주간보고 만들어줘: 1. 전체 현황 / 2. 팀별 실적 / 3. 특이사항 '주간보고_4월4주차.md'로 저장"
✅ 표 형식 보고서가 .md로 떨어지면 OK

10 실습 2-C · 10명 개인화 메일 대기

사용 파일: 📧 메일_발송_명단.xlsx

"메일_발송_명단.xlsx 열어서 10명에게 보낼 정산 안내 메일을 각각 개인화해서 만들어줘: - 담당자명 호명 - 공급가액·VAT·합계금액 표기 - 세금계산서 15일 발행 요청 포함 '정산메일_초안.md'에 10건 모두 저장"
✅ 10명분 각자 다른 이름·금액으로 작성된 메일 .md → 완료

11 실습 3-A · 웹 대시보드 대기

사용 파일: 📋 팀원_업무현황.xlsx

"이 파일로 Streamlit 대시보드 만들어줘. 팀별 진행률 막대그래프 + 지연 태스크 빨간색"
💡 "streamlit 없다"고 하면 → "streamlit 설치해줘" 한 줄 추가
✅ 브라우저에 막대그래프 떴으면 성공 🎉

12 실습 3-B · 회의록 → Action Item 대기

회의 내용을 그대로 터미널에 붙여넣기 (다운로드 파일 없음)

"이 회의 내용에서 Action Item만 표로 뽑아줘. 담당자/마감일/우선순위 컬럼 포함. '회의_액션아이템.md'로 저장"

응용 (선택)

"이걸 Dooray 태스크로 자동 등록해줘"
✅ Dooray 웹에서 새 태스크 보이면 완료

13 마무리 — CLAUDE.md 만들기 대기

내일부터 매번 설명 안 하려면 내 정보·규칙을 한 번 저장해두세요.

"내 작업 폴더에 CLAUDE.md 만들어줘: - 이름: 홍길동 - 부서: 경영관리팀 - 보고서 규칙: 천 단위 콤마, 전월 대비 필수 - 메일 톤: 정중한 비즈니스 다음부터 자동 적용되게."
✅ 이후 "이번 주 주간보고" 한 줄만 쳐도 위 규칙 자동 적용 — 10줄 → 1줄

14 과제 제출 필수 대기

오늘 설치한 Dooray MCP를 그대로 활용 — Claude Code에서 한 줄로 끝.

  1. 강의 사이트 접속 → 실습 3 메뉴
  2. "제출 프롬프트 생성" 버튼 클릭
  3. 나타난 프롬프트 전체 복사
  4. Claude Code 창에 그대로 붙여넣고 Enter
  5. ✅ Dooray에 과제 태스크 자동 생성 완료
